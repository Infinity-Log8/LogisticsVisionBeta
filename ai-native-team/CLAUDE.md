# AI-Native Team — Project Memory

## Project Overview
AI-native cross-functional team (Design, Engineering, Marketing) operating as a single unit.
Team size: 8-12 senior people. AI is a core multiplier, not a bolt-on. Every workflow,
document, and process here assumes AI is part of how work gets done — and that humans
remain accountable for judgment, taste, and final calls.

## Team Structure
- Design (2-3): Senior Product Designer, Visual/Brand Designer, UX Researcher (part-time)
- Engineering (3-5): Tech Lead/Architect, 2-3 Mid-Senior Engineers, DevOps/Infra Engineer
- Marketing (2-3): Strategic Marketer, Content/Growth Marketer, Data/Performance Marketer (optional)
- Cross-functional: Product Manager

## Project Structure
- `/team/` — org chart, role profiles, hiring criteria
- `/workflows/` — weekly rhythm, feature lifecycle, content pipeline, design system
- `/tooling/` — AI stack definition, shared prompt library
- `/quality/` — quality gates, risk register
- `/economics/` — budget model, ROI framework
- `/getting-started/` — 30-60-90 plan, first sprint template
- `/memory/` — session logs, decision log, context snapshots
