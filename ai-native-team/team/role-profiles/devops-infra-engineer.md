# Infrastructure / DevOps Engineer

**Reports to:** Tech Lead
**Function:** Engineering

## Core Responsibilities
- Own CI/CD pipelines, deployment automation, and infrastructure-as-code
- Set up and own monitoring, alerting, and on-call rotation
- Lead incident response and post-mortems
- Manage cloud spend, security baselines, and dependency hygiene
- Build internal developer platform — reduce the time from commit to production

## AI-Augmented Responsibilities
- **AI handles:** analyzing logs, generating runbooks from incidents, drafting
  Terraform/IaC modules, writing post-mortem first drafts, summarizing alert noise
  patterns, suggesting cost optimizations from billing data
- **Human judges:** whether a fix addresses the root cause or just the symptom, when
  a runbook needs a human escalation step, where to spend the cost-optimization
  budget, when AI's IaC suggestion bakes in a security mistake

## Key Skills to Hire For
- Has run production at non-trivial scale for at least one team
- Comfortable with at least one major cloud (AWS, GCP, Azure) end-to-end
- Strong scripting + shell fluency — can build a tool in 30 minutes
- Calm during incidents — communicates clearly while debugging
- Security-aware without being security-theater

## Interview Signals (AI-native vs. traditional)
- **Strong:** Can describe a recent incident, the actual root cause, and the system
  change that prevents recurrence (not just "we'll be more careful")
- **Strong:** Has used AI to summarize a noisy alerting system and reduce false
  positives
- **Strong:** Has an opinion on observability that goes beyond "we use Datadog"
- **Weak:** Conflates DevOps with "I know Kubernetes" — no broader systems thinking
- **Weak:** Has never been on-call, or describes on-call only as suffering

## Compensation Band (US, mid-range senior IC)
- Base: $185K - $230K
- Equity: 0.2% - 0.6% (early-stage)
- Total comp target: $250K - $330K
